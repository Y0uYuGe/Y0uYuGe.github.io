```c
#include<stdio.h>

int main()
{
    unsigned int a,b,c;
    c=0x87654321;
    a=(c<<16)>>16;
    b=c>>16;
    printf("a: 0x%x\n",a);
    printf("b: 0x%x\n",b);
}
```

```bash
.section .data
number:     .int 0x87654321
lowBits:    .int 0
highBits:   .int 0

.section .text
.globl _start
_start:
    movl number, %eax
    sall $16,%eax
    shrl $16, %eax
    movl %eax, lowBits

    movl number, %eax
    shrl $16, %edx
    movl %edx, highBits

    movl $1, %eax
    int $0x80
```

```bash
.macro set_struct s,a,b
    movl \a,\s
    movl \b,\s+4
.endm

.macro get_struct s,a,b
    movl \s,\a
    movl \s+4,\b
.endm

.section .data

stru:  
    .int 0
    .int 0

.section .text
.globl _start
_start:
    nop

.foo:
    movl $0x12345678, %eax
    movl $0x87654321, %edx
    set_struct stru,%eax,%edx
    movl $0, %eax
    movl $0, %edx
    get_struct stru,%eax,%edx
    ret
```


### register
在C语言中，register关键字用于告诉编译器，变量a应尝试在CPU寄存器中存储，而不是RAM中。这在原理上可以让变量a的访问和操作更快，因为CPU寄存器的数据可以直接被处理，无需花费时间访问 RAM。然而，此建议通常对现代优化编译器没有实际影响，因为编译器通常会自动选择将哪些变量存储在寄存器中。

上述语句声明了一个叫做a的无符号整型变量，尝试将它存储在寄存器中。

请注意，register关键字只是一个建议，编译器可以选择忽略它。此外，register变量不能取地址，因为它们可能不存在内存中，从而没有内存地址可供引用。

然而在现代编程中，使用register关键字现在已经很罕见，优化编译器已经足够聪明，可以自动决定哪些变量应存储在寄存器中，以获得最佳性能
###  .macro macro_name [parameter1, parameter2, ...] ; 汇编指令 .endm

```bash
.macro macro_name [parameter1, parameter2, ...] ; 汇编指令 .endm
```

- macro_name ：这是你定义的宏的名称，当你调用宏的时候会用到。
- parameter1, parameter2, ... ：这些是传递给宏的参数。在宏体内，你可以使用反斜杠 followed by the parameter name (\parameter1, \parameter2, etc.) 来引用这些参数。
- 在.macro和.endm指令之间的所有行都被视为宏的定义，当宏被调用时，这些行就会被替代宏的调用..macro macro_name [parameter1, parameter2, ...] ;
